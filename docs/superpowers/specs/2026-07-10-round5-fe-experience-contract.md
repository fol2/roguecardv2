### Canonical shape profiles

Canonical profile rectangles and final resolved coordinates are stage pixels. An unqualified percentage in a P6 screen row is relative to that profile's Body rectangle. For each supplied percentage component, with Body rectangle `B`, resolve `stageX = B.x + B.w × x% / 100`, `stageY = B.y + B.h × y% / 100`, `stageW = B.w × w% / 100` and `stageH = B.h × h% / 100`. Merely naming a layer header, action, choice or rail does not change the default region without one of those explicit qualifiers. Safe-area inset expressions and the inherited Title-version anchors bypass percentage conversion and remain literal stage-edge offsets, including their listed safe-right and safe-bottom additions. Each profile reserves its listed safe composition rectangles; no authored layer extends outside the selected region.

### Locale fit and inherited Title-version chrome

Every P6 text surface receives a stable locale key and PE-resolved text. FE renders that resolved text unchanged and owns only its visual hierarchy and fit: display titles have a two-line maximum, and action labels have a two-line maximum inside their existing 44px-or-larger target. Body copy has no numeric line cap: it wraps without ellipsis inside its resolved bounded body-copy region. The nearest semantic copy scroll host is the only element allowed to scroll and becomes scrollable exactly when its DOM `scrollHeight > clientHeight`; the stage, header and action rail remain fixed. Player choices, exact numerals and accessibility text are never ellipsised, truncated or rewritten. A key that resolves to itself is a PE missing-key defect, not copy for FE to hide or rewrite.

All visual-regression baseline captures assert active locale `en` before capture. One separate non-baseline synthetic-locale overflow review may expand PE-supplied resolved strings to diagnose clipping, overlap, wrapping and scroll containment; it neither changes a baseline nor proves translation readiness. This contract adds no language picker, second locale, RTL support or font-coverage claim.

The two inherited version states are base Title chrome, not extra P6 or Phase 2 rows. Both are right-aligned, single-line diagnostics with intrinsic width capped at 46% of the stage. The default uses `textDim` at opacity0.70 and letter spacing0.06em; the transient detail uses `textDim` at opacity0.85 and letter spacing0.04em. Neither has a frame, fill, sheen, glow or filter. Phone portrait and phone landscape use 10px/12px; pad portrait, pad landscape and desktop landscape use 11px/13px. The action column and Title-statistics box may not intersect either version box and retain an 8px edge gap; the lower-left Rose medallion retains a 24px horizontal gap. If statistics enter the same vertical band, their inline end stops before the version box rather than moving the inherited version anchor.

Full and LITE show the same static terminal chrome. REDUCED renders the same terminal coordinates without transition: `title-version-default` has the default label visible and transient detail hidden; `title-version-debug` has both visible after PE supplies that state. Each REDUCED terminal records `motion=reduced`, `outcome=settled`. Version visibility changes use 0ms and no FE-authored easing; PE owns the transient state's timing and accessibility announcement.
