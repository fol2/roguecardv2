# bfuiedit — UI Chrome Layout Editor — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a forever-tunable, per-stage-shape combat chrome layout (`?bfuiedit=1`) so energy / lantern / end-turn / piles / top HUD can be edited live and saved like `?bfedit=1`.

**Architecture:** Pure data `UIC` in `src/ui-chrome-layout.js` + Node-safe resolver `src/uic.js` (mirror `BF` / `battlefield.js`). Dev editor mutates via `_setUIC` and saves through `POST /__bfui-save`. `ui.js` applies resolved stage-px styles; CSS keeps visuals and acts as fallback only.

**Tech Stack:** Vite 8 ESM, vanilla JS, existing `npm test` (`test/test_engine.js`), optional Playwright later.

**Sketch (locked decisions):** `.superpowers/sdd/bfuiedit-sketch.md`

## Global Constraints

- `src/ui-chrome-layout.js` and `src/uic.js` import nothing (except layout ← resolver) and stay Node-importable. Never import `audio.js`, `stage.js`, or DOM modules from them.
- Do **not** reuse / overwrite `src/ui-chrome.js` (existing asset helpers). Resolver file is `src/uic.js`.
- Shape keys = stage shapes: `phone-portrait`, `phone-landscape`, `pad-portrait`, `pad-landscape`, `desktop-landscape`.
- Anchors replicate CSS: widgets keep `left` **or** `right` (+ `bottom`). Do not force right-side chrome onto left-only storage.
- Safe-area is **baked** into layout numbers. Apply writes plain `px` — no `+ var(--sa*)` at apply-time.
- Top HUD is a **unit** (`height` / `scale` on `.hud-bar`), not per-icon drag.
- Editor loads only via `import.meta.env.DEV && ?bfuiedit=1`. Save endpoint `apply: 'serve'` only.
- Engine / hand fan / art regen stay untouched.
- Commit only if the owner asks.

### Data shape (v1)

```js
// src/ui-chrome-layout.js — all numbers STAGE px; safe-area baked in.
export const UIC = {
  base: {
    energy:  { left: 24,  bottom: 148 },
    lantern: { left: 42,  bottom: 262, w: /* from CSS */, h: /* from CSS */ },
    endTurn: { right: 22, bottom: 128, w: 120, h: 120 },
    draw:    { left: 16,  bottom: 14,  w: /* … */, h: /* … */ },
    discard: { right: 22, bottom: 14,  w: /* … */, h: /* … */ },
    ashes:   { right: 132, bottom: 14, w: /* … */, h: /* … */ },
    hud:     { height: /* bar content height */, scale: 1 },
  },
  shapes: {
    'phone-portrait': { /* seed from ≤740 @container block */ },
    'phone-landscape': { /* seed from landscape height rules */ },
    'pad-portrait': { /* seed from ≤1100 block where distinct */ },
    'pad-landscape': {},
    'desktop-landscape': {},
  },
};
```

Seed from `src/styles.css` **content** offsets (the number before `+ var(--sa*)`), not the live env inset. Right-anchored keys stay `right`.

### Apply contract

For each widget, set only the edges present in data (`left` xor `right`, `bottom`, optional `w`/`h`). Clear the opposite edge (`el.style.right = ''` when applying `left`, and vice versa). For `hud`, apply `height` and `transform: scale(...)` (or equivalent CSS var) on `.hud-bar` as a unit; do not reposition individual `.icon-btn`s.
