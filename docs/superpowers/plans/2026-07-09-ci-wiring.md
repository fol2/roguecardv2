# CI Wiring (Round 5 Phase 0) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** GitHub Actions runs the full quality kit (`npm test` + `npm run build` + Playwright) on every PR and push to main, with a nightly non-gating perf job, failure artifacts, and refreshed visual baselines so the gate starts green.

**Architecture:** Three workflow files under `.github/workflows/`: `ci.yml` (gating: unit+build+committed-dist job and e2e job), `perf.yml` (nightly cron + manual dispatch, non-gating by design — CI runner fps numbers are noisy), and `update-baselines.yml` (manual dispatch that regenerates only the linux visual suite on a real ubuntu runner and uploads it as an artifact, because the reference machine is darwin and cannot produce linux pixels). CI runs the source-writing editor test once first, then the long random-agent playback alone with one worker and the production reduced-motion branch, then every remaining test with the deliberate two-worker/no-retry contract; this preserves all executable coverage without letting a writer or animation-heavy page contaminate another. The random-agent/main/visual/perf lanes use `--no-deps`. Visual baselines are refreshed darwin locally and linux from a PR-run artifact, so the PR is green before merge.

**Tech Stack:** GitHub Actions on pinned `ubuntu-24.04` (full-SHA pinned `actions/checkout@v7`, `actions/setup-node@v6`, `actions/upload-artifact@v7`), Playwright 1.61 (`@playwright/test` devDependency, config in `playwright.config.js`), `gh` CLI for PR/dispatch/branch-protection.

**Spec:** `docs/superpowers/specs/2026-07-09-canvas-ui-shipfront-design.md`, heading `Testing & QA → Phase 0 — CI` (currently §8; use the heading because the parallel design amendment can renumber sections).

## Revision after the first PR run (2026-07-09)

PR #8's first run (`29050825682`) proved that the original plan was incomplete: `unit` passed, but `e2e` had 32 failures (16 expected stale linux screenshots and 16 real non-visual failures). The non-visual failures have four test/infrastructure root causes, none in application behaviour:

1. `geometry.spec.js` still treated raw transparent image-box bottoms as literal feet. The runtime now deliberately composes per-character `footY` and per-formation `y`; the test must calculate the visual foot using the same Node-safe battlefield resolver.
2. Two mocked battlefield-editor saves intercepted `/__bf-save` but allowed the preceding `/__char-save` to write `src/char-meta.js`. Vite HMR then navigated unrelated parallel pages. The dedicated disk test also waited for an accidental page load even though the editor contract is now soft HMR.
3. Once the feet assertion was repaired, the focused geometry run exposed a second stale raw-box assumption: `.sl-ledge` top was treated as the painted lip. Commit `389f4be` intentionally introduced oversized/zoomed, alpha-transparent ground plates and the runtime clamps their CSS bottom to the stage. Geometry must validate the resolved rendered box formula and coverage of the ground line; visual baselines own the actual painted pixels.
4. A focused two-worker editor run exposed a locator race in all three drag tests: layer images decode asynchronously, and `syncOverlays()` deliberately replaces every `.bf-box` so alpha-aware bounds can be drawn. A separate `waitFor()` followed by `boundingBox()` can land between detach/recreate; an atomic centre read alone is still insufficient because replacement can occur before pointerdown. First await all combat layer `img.decode()` calls plus two animation frames (letting the final load callback rebuild the overlay), then read the live locator centre atomically. Do not add retries or change the editor.

The repaired full darwin suite then exposed a fifth harness defect: `waitMapSettled()` could take both of its 220 ms samples before the first map-mode frame and mistake “not started” for “settled”. Landscape repetition produced 2 passes / 1 failure, then five failures at one worker with roughly 4,471–4,620 differing pixels. Timing probes also showed why a naive “unchanged transform” wait is wrong: `scene3d.js` applies a time-based camera roll indefinitely. The stable camera-state signal is each node's projected `scale` plus inline `opacity` (distance and vertical focus, invariant to roll). Across desktop, portrait, and landscape this signal stopped changing by about 4.92 s, but had pre-final plateaux up to about 658 ms; therefore the harness must observe a post-show signal change and then require 800 ms of quiet.

The first condition-based implementation produced five landscape actuals that Playwright's own PNG comparator considered identical to one another, while every one differed from the old darwin baseline by about 3,735–3,782 pixels. Desktop and portrait still matched. Visual inspection confined the delta to the projected bottom lanterns: the old landscape baseline canonised an in-flight camera position. After that actual-to-actual stability proof, replacing only `map-landscape-darwin.png` is part of migrating the timing contract, not a way to conceal a flake; all 70 repetitions must pass against the promoted image.

The audit also found that `test:e2e:update` and `test:e2e:perf` unintentionally ran the source-writing `bfeditor-disk` dependency, and that committed `dist/` had drifted behind source. Tasks 3A–3B close all identified gaps before PR #8 is allowed to merge.

```yaml
          test -z "$(git status --porcelain --untracked-files=all)"
```

```js
const MAP_SETTLE_QUIET_MS = 800;
const MAP_SETTLE_QUIET_FRAMES = 20;
const MAP_SETTLE_TIMEOUT_MS = 12_000;

/** Wait for the map dolly's roll-invariant distance/focus signal to settle. */
async function showMapAndWaitSettled(page) {
  await page.evaluate(() => {
    window.__visualMapSettle = {
      initial: null,
      last: null,
      moved: false,
      quietSince: 0,
      quietFrames: 0,
    };
    window.spirebound.show('map');
  });
  try {
    await page.waitForFunction(({ quietMs, quietFrames }) => {
      const nodes = [...document.querySelectorAll('.mnode')];
      const edges = [...document.querySelectorAll('.medge')];
      const values = nodes.map((node) => {
        const transform = node.getAttribute('transform') || '';
        const scale = transform.match(/scale\(([^)]+)\)/)?.[1];
        return scale && node.style.opacity ? `${scale}|${node.style.opacity}` : null;
      });
      const edgeOpacities = edges.map((edge) => edge.style.opacity || null);
      if (!values.length || values.some((value) => !value) || edgeOpacities.some((value) => !value)) {
        return false;
      }
      const current = [...values, ...edgeOpacities].join(';');
      const state = window.__visualMapSettle;
      const now = performance.now();
      if (state.initial == null) {
        state.initial = current;
        state.last = current;
        state.quietSince = now;
        return false;
      }
      if (current !== state.last) {
        state.moved ||= current !== state.initial;
        state.last = current;
        state.quietSince = now;
        state.quietFrames = 0;
        return false;
      }
      state.quietFrames += 1;
      return state.moved
        && state.quietFrames >= quietFrames
        && now - state.quietSince >= quietMs;
    }, {
      quietMs: MAP_SETTLE_QUIET_MS,
      quietFrames: MAP_SETTLE_QUIET_FRAMES,
    }, { polling: 'raf', timeout: MAP_SETTLE_TIMEOUT_MS });
  } finally {
    await page.evaluate(() => { delete window.__visualMapSettle; }).catch(() => {});
  }
}
```
