# CI Wiring (Round 5 Phase 0) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** GitHub Actions runs the full quality kit (`npm test` + `npm run build` + Playwright) on every PR and push to main, with a nightly non-gating perf job, failure artifacts, and refreshed visual baselines so the gate starts green.

**Architecture:** Three workflow files under `.github/workflows/`: `ci.yml` (gating: unit+build job and e2e job), `perf.yml` (nightly cron + manual dispatch, non-gating by design — CI runner fps numbers are noisy), and `update-baselines.yml` (manual dispatch that regenerates the linux visual baselines on a real ubuntu runner and uploads them as an artifact, because the reference machine is darwin and cannot produce linux pixels). Visual baselines are stale (last captured at PR #6, ~55 combat-chrome commits ago) and are refreshed as part of this plan — darwin locally, linux via the dispatch workflow after merge.

**Tech Stack:** GitHub Actions (`actions/checkout@v4`, `actions/setup-node@v4`, `actions/upload-artifact@v4`), Playwright 1.61 (`@playwright/test` devDependency, config in `playwright.config.js`), `gh` CLI for PR/dispatch/branch-protection.

**Spec:** `docs/superpowers/specs/2026-07-09-canvas-ui-shipfront-design.md` §6 (Phase 0 — CI).

## Global Constraints

- **Do not touch `src/**`** — Progressive Delivery Phase 2 is executing in parallel on `src/ui.js`. This plan may only create/modify: `.github/**`, `test/e2e/visual.spec.js-snapshots/**`, `AGENTS.md`, `docs/**`.
- **Do not modify `playwright.config.js`.** `workers: 2`, `retries: 0`, `trace: 'off'` are deliberate (WebGL constraints, flake-must-surface policy). CI adds no retries anywhere.
- Node version on CI: **24** (matches the reference machine, v24.2.0).
- Gating job names are pinned to exactly `unit` and `e2e` (branch protection references them by name).
- The perf workflow is **non-gating**: its measurement step uses `continue-on-error: true`; numbers are read from the job log.
- Work happens on branch `ci/p0-wiring`; the repo convention allows direct commits to main only for the post-merge baseline commit in Task 5.
- Every commit message ends with the repo's standard `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>` trailer.

### Task 2: Gating workflow `.github/workflows/ci.yml`

**Files:**
- Create: `.github/workflows/ci.yml`
- Modify: `AGENTS.md` (one CI line in the Commands section)

**Interfaces:**
- Consumes: npm scripts `test`, `build` from `package.json`; `playwright.config.js` webServer (auto-starts `npm run dev`).
- Produces: check runs named exactly `unit` and `e2e` (Task 7 branch protection references these strings).

- [ ] **Step 1: Write the workflow**

Create `.github/workflows/ci.yml`:

```yaml
name: ci

on:
  push:
    branches: [main]
  pull_request:

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  unit:
    name: unit
    runs-on: ubuntu-latest
    timeout-minutes: 15
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
          cache: npm
      - run: npm ci
      # engine unit checks + asset manifest + 300-run monte-carlo
      - run: npm test
      # production bundle must build clean
      - run: npm run build

  e2e:
    name: e2e
    runs-on: ubuntu-latest
    timeout-minutes: 45
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
          cache: npm
      - run: npm ci
      - run: npx playwright install --with-deps chromium
      # full kit: stage/geometry/battle/visual/rewards/bfeditor.
      # perf.spec self-skips (PERF env unset). Baselines compare *-linux.png.
      - run: npx playwright test
      - uses: actions/upload-artifact@v4
        if: failure()
        with:
          name: playwright-failures
          path: test-results/
          retention-days: 14
```

### Task 3: Non-gating perf workflow and baseline-update workflow

**Files:**
- Create: `.github/workflows/perf.yml`
- Create: `.github/workflows/update-baselines.yml`

**Interfaces:**
- Consumes: npm script `test:e2e:perf` (`PERF=1 playwright test perf --workers=1`).
- Produces: artifact `visual-baselines-linux` (Task 5 downloads it); manually dispatchable workflows named `perf` and `update-baselines` (dispatch requires the files to exist on the default branch — hence Task 5 runs after merge).

- [ ] **Step 1: Write the perf workflow**

Create `.github/workflows/perf.yml`:

```yaml
name: perf

on:
  schedule:
    - cron: "0 3 * * *"
  workflow_dispatch:

jobs:
  perf:
    name: perf
    runs-on: ubuntu-latest
    timeout-minutes: 25
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
          cache: npm
      - run: npm ci
      - run: npx playwright install --with-deps chromium
      # Non-gating by spec: CI runner fps numbers are noisy (software WebGL).
      # The measured numbers are recorded in this step's log, never enforced.
      - run: npm run test:e2e:perf
        continue-on-error: true
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: perf-results
          path: test-results/
          retention-days: 30
          if-no-files-found: ignore
```

- [ ] **Step 2: Write the baseline-update workflow**

Create `.github/workflows/update-baselines.yml`:

```yaml
name: update-baselines

on:
  workflow_dispatch:

jobs:
  update:
    name: update
    runs-on: ubuntu-latest
    timeout-minutes: 45
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
          cache: npm
      - run: npm ci
      - run: npx playwright install --with-deps chromium
      # Regenerate ONLY the visual suite's snapshots on a real ubuntu runner;
      # the darwin reference machine cannot produce linux pixels.
      - run: npx playwright test visual --update-snapshots
      - uses: actions/upload-artifact@v4
        with:
          name: visual-baselines-linux
          path: test/e2e/visual.spec.js-snapshots/*-linux.png
          retention-days: 7
```

- [ ] **Step 3: Sanity-check both files parse**

```bash
npx --yes yaml-lint .github/workflows/perf.yml
npx --yes yaml-lint .github/workflows/update-baselines.yml
```

Expected: both valid.

- [ ] **Step 4: Commit**

```bash

git commit -m "$(cat <<'EOF'
Add nightly non-gating perf and manual baseline-update workflows.

Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>
```
