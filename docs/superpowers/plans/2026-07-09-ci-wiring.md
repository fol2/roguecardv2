# CI Wiring (Round 5 Phase 0) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** GitHub Actions runs the full quality kit (`npm test` + `npm run build` + Playwright) on every PR and push to main, with a nightly non-gating perf job, failure artifacts, and refreshed visual baselines so the gate starts green.

**Architecture:** Three workflow files under `.github/workflows/`: `ci.yml` (gating: unit+build+committed-dist job and e2e job), `perf.yml` (nightly cron + manual dispatch, non-gating by design — CI runner fps numbers are noisy), and `update-baselines.yml` (manual dispatch that regenerates only the linux visual suite on a real ubuntu runner and uploads it as an artifact, because the reference machine is darwin and cannot produce linux pixels). The full suite keeps its deliberate two-worker/no-retry contract; source-writing editor coverage runs once as a dependency, while single-purpose visual/perf commands use `--no-deps`. Visual baselines are refreshed darwin locally and linux from a PR-run artifact, so the PR is green before merge.

**Tech Stack:** GitHub Actions on pinned `ubuntu-24.04` (full-SHA pinned `actions/checkout@v7`, `actions/setup-node@v6`, `actions/upload-artifact@v7`), Playwright 1.61 (`@playwright/test` devDependency, config in `playwright.config.js`), `gh` CLI for PR/dispatch/branch-protection.

**Spec:** `docs/superpowers/specs/2026-07-09-canvas-ui-shipfront-design.md`, heading `Testing & QA → Phase 0 — CI` (currently §8; use the heading because the parallel design amendment can renumber sections).

## Revision after the first PR run (2026-07-09)

PR #8's first run (`29050825682`) proved that the original plan was incomplete: `unit` passed, but `e2e` had 32 failures (16 expected stale linux screenshots and 16 real non-visual failures). The non-visual failures have four test/infrastructure root causes, none in application behaviour:

1. `geometry.spec.js` still treated raw transparent image-box bottoms as literal feet. The runtime now deliberately composes per-character `footY` and per-formation `y`; the test must calculate the visual foot using the same Node-safe battlefield resolver.
2. Two mocked battlefield-editor saves intercepted `/__bf-save` but allowed the preceding `/__char-save` to write `src/char-meta.js`. Vite HMR then navigated unrelated parallel pages. The dedicated disk test also waited for an accidental page load even though the editor contract is now soft HMR.
3. Once the feet assertion was repaired, the focused geometry run exposed a second stale raw-box assumption: `.sl-ledge` top was treated as the painted lip. Commit `389f4be` intentionally introduced oversized/zoomed, alpha-transparent ground plates and the runtime clamps their CSS bottom to the stage. Geometry must validate the resolved rendered box formula and coverage of the ground line; visual baselines own the actual painted pixels.
4. A focused two-worker editor run exposed a locator race in all three drag tests: layer images decode asynchronously, and `syncOverlays()` deliberately replaces every `.bf-box` so alpha-aware bounds can be drawn. A separate `waitFor()` followed by `boundingBox()` can land between detach/recreate; an atomic centre read alone is still insufficient because replacement can occur before pointerdown. First await all combat layer `img.decode()` calls plus two animation frames (letting the final load callback rebuild the overlay), then read the live locator centre atomically. Do not add retries or change the editor.

The audit also found that `test:e2e:update` and `test:e2e:perf` unintentionally ran the source-writing `bfeditor-disk` dependency, and that committed `dist/` had drifted behind source. Tasks 3A–3B close all four gaps before PR #8 is allowed to merge.

```js
async function measure(page) {
  await page.waitForFunction(() => window.__probe?.geometry());
  const g = await page.evaluate(() => {
    const geometry = window.__probe.geometry();
    if (!geometry) return null;
    return {
      ...geometry,
      act: window.spirebound.S.run.act,
      aspect: window.spirebound.S.run.aspect,
      enemyLayoutKeys: window.spirebound.S.cb.enemies.map((en) => en.key),
    };
  });
  expect(g, 'probe.geometry() should measure a live combat').not.toBeNull();
  return g;
}
```

```js
function assertGrounded(g, label) {
  const layout = bfResolve(g.stage.shape, g.act);
  const aspect = ASPECTS[g.aspect];
  expect(aspect, `${label}: live aspect exists`).toBeTruthy();
  const hero = bfActor('heroes', aspect.id);
  expect(g.heroArtBottom, `${label}: hero art bottom exists`).not.toBeNull();
  const heroExpectedBottom = g.groundY - bfHeroY(layout) - hero.footY;
  expect(Math.abs(g.heroArtBottom - heroExpectedBottom),
    `${label}: hero art box honours authored lift and foot correction`)
    .toBeLessThanOrEqual(FEET_TOL);

  const slots = bfSlots(layout, g.enemyLayoutKeys.length);
  g.enemyArtBottoms.forEach((bottom, i) => {
    if (bottom == null) return;
    const expectedBottom = g.groundY
      - (slots[i]?.y ?? 0)
      - bfEnemyFootY(slots[i], g.enemyLayoutKeys[i]);
    expect(Math.abs(bottom - expectedBottom),
      `${label}: enemy ${i} art box honours authored lift and foot correction`)
      .toBeLessThanOrEqual(FEET_TOL);
  });

  if (g.slLedgeTop != null) {
    const ledge = layout.layers.ledge;
    const cssBottom = Math.max(0,
      layout.groundY + layout.ledgeLip - ledge.h + ledge.y);
    const expectedTop = g.stage.h - cssBottom - ledge.h * ledge.zoom;
    expect(Math.abs(g.slLedgeTop - expectedTop),
      `${label}: ledge box follows authored placement`)
      .toBeLessThanOrEqual(FEET_TOL);
    expect(layout.ledgeLip).toBeGreaterThanOrEqual(LEDGE_LIP_MIN);
    expect(layout.ledgeLip).toBeLessThanOrEqual(LEDGE_LIP_MAX);
    expect(g.slLedgeTop).toBeLessThanOrEqual(g.groundY + FEET_TOL);
    expect(g.slLedgeTop + ledge.h * ledge.zoom)
      .toBeGreaterThanOrEqual(g.groundY - FEET_TOL);
  }

  if (g.seamY != null) {
    expect(Math.abs(g.groundY - g.seamY),
      `${label}: logical glow seam stays on the ground line`)
      .toBeLessThanOrEqual(FEET_TOL);
  }
}
```
